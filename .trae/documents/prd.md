## 1. 产品概述

医疗器械不良事件上报系统，用于医疗机构对医疗器械使用过程中发生的不良事件进行登记、上报、整改跟踪和归档管理，确保合规性和可追溯性。
- 目标用户：医院设备科、医疗器械监管人员、质量管理人员
- 核心价值：实现不良事件从发现到闭环整改的全生命周期管理，满足法规对严重事件24小时上报的强制要求，保障患者隐私

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 上报员 | 系统分配 | 登记事件、提交上报、查看本人事件 |
| 审核员 | 系统分配 | 审核事件、分配整改任务、归档 |
| 管理员 | 系统分配 | 全部权限、用户管理、数据统计 |

### 2.2 功能模块

1. **事件列表页**：事件筛选、状态标签、快速搜索、分页浏览
2. **事件登记页**：填写事件信息、严重等级选择、患者信息录入、设备信息关联
3. **事件详情页**：事件完整信息、状态流转历史、整改记录、脱敏展示提示
4. **整改跟踪页**：整改任务列表、整改进度、关闭整改、归档操作

## 3. 核心流程

医疗器械不良事件上报主流程：事件登记 → 上报审核 → 整改分配 → 整改执行 → 整改关闭 → 归档闭环

异常分支：
- 严重事件登记时，系统拦截提示必须在24小时内完成上报
- 患者信息在列表和详情中脱敏展示，并显示脱敏提示
- 整改未全部关闭时，归档操作被拦截并提示

```mermaid
flowchart TD
    "事件登记" --> "选择严重等级"
    "选择严重等级" --> "严重/特别严重？"
    "严重/特别严重？" -- 是 --> "弹出24小时上报时限提示"
    "严重/特别严重？" -- 否 --> "提交事件"
    "弹出24小时上报时限提示" --> "确认后提交事件"
    "提交事件" --> "上报审核"
    "上报审核" --> "审核通过？"
    "审核通过？" -- 否 --> "驳回修改"
    "驳回修改" --> "事件登记"
    "审核通过？" -- 是 --> "分配整改任务"
    "分配整改任务" --> "整改执行"
    "整改执行" --> "整改关闭"
    "整改关闭" --> "全部整改已关闭？"
    "全部整改已关闭？" -- 否 --> "提示存在未关闭整改"
    "全部整改已关闭？" -- 是 --> "归档"
    "归档" --> "闭环"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深青色 (#0F766E) — 医疗行业专业感
- 辅助色：琥珀色 (#D97706) — 警示与强调
- 危险色：红色 (#DC2626) — 严重事件/拦截提示
- 背景色：浅灰 (#F8FAFC) 卡片式布局
- 按钮风格：圆角 (rounded-lg)、带悬浮阴影
- 字体：Noto Sans SC (中文主字体)
- 布局：左侧导航栏 + 右侧内容区，顶部状态栏
- 图标：lucide-react 医疗主题图标

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 事件列表页 | 筛选区 | 下拉选择器、日期范围选择、搜索输入框 |
| 事件列表页 | 事件表格 | 条纹表格、状态标签徽章、脱敏姓名、分页器 |
| 事件登记页 | 表单区 | 分组卡片表单、下拉选择、日期选择、文本域、严重等级高亮提示 |
| 事件登记页 | 拦截弹窗 | 模态对话框、倒计时提示、确认按钮 |
| 事件详情页 | 信息卡 | 分栏信息展示、脱敏标识图标、状态时间线 |
| 事件详情页 | 操作区 | 状态流转按钮组、归档拦截弹窗 |
| 整改跟踪页 | 任务列表 | 看板卡片、进度条、截止日期高亮 |

### 4.3 响应式设计

桌面优先设计，平板和移动端自适应：表格在小屏转为卡片布局，侧边栏可折叠
